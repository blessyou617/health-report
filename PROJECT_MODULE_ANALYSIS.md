# 项目功能与模块分析（当前版本）

## 1. 项目整体定位

该项目是一个“体检报告 AI 解读”多端系统，包含：

- `api/`：FastAPI 后端，负责鉴权、报告上传、AI 分析、追问问答、支付与语音生成。
- `app/`：React Native + Expo 移动端，负责问卷、报告查看、语音播放与分享入口。
- `components/`：微信小程序风格的体检报告组件库，偏展示层，可用于小程序页面快速搭建。

## 2. 核心业务流程

典型流程可以概括为：

1. 用户通过微信登录拿到后端 JWT。
2. 上传体检报告文件（PDF/JPG/PNG）。
3. 提交生活方式问卷数据。
4. 触发 AI 分析并轮询状态。
5. 查看报告摘要与异常项。
6. 支付后解锁完整能力（完整报告、TTS、追问等）。
7. 可进行最多 2 次报告追问。
8. 可为报告或问答生成语音。

## 3. 后端模块（`api/`）

### 3.1 接口路由层（Routers）

- `app/routers/api.py`
  - 认证：`/auth/login`
  - 报告：上传 `/upload/report`、问卷 `/questionnaire/submit`、分析 `/analyze/report/{report_id}`、详情与状态查询
  - 追问：`/qa/ask`、`/qa/status/{report_id}`，并在后端执行“是否解锁 + 次数限制”校验
  - 语音：`/tts/generate`、`/tts/report/{report_id}`、`/tts/qa/{qa_id}`
- `app/routers/payment.py`
  - 支付单创建、支付状态查询、支付回调、前端轮询验证支付结果

特点：路由层承担参数校验、鉴权、所有权校验与业务服务编排。

### 3.2 领域模型层（Models）

- `User`：微信用户主体（`openid` 唯一）。
- `Report`：报告主记录，包含文件信息、问卷、分析结果、支付解锁状态、追问计数与状态机字段。
- `Payment`：支付记录（订单号、金额、状态、支付渠道流水）。
- `QAHistory`：追问历史。
- 枚举：`ReportStatus`、`PaymentStatus`。

特点：`Report` 同时承载“报告状态机 + 付费解锁 + 追问配额”关键状态，是核心聚合。

### 3.3 服务层（Services）

- `ai.py`
  - 调 OpenAI 多模态能力分析报告文件并返回结构化 JSON（总结/异常/正常/建议）。
- `qa.py`
  - 基于报告分析、问卷信息与历史追问构建上下文后生成回答。
  - 强制最多 2 次追问，并以数据库历史条数作为真相源。
- `tts.py`
  - 调 OpenAI TTS 生成 MP3；可生成“报告摘要语音”和“问答语音”。
- `payment.py`
  - 微信支付下单、签名参数、回调验签（示例级简化）、支付成功后原子化解锁报告。
- `file.py`
  - 上传文件类型校验、命名与本地落盘。
- `wechat.py`
  - 微信 `code2session` 登录换取 `openid`。

特点：服务层按能力拆分清晰，便于独立演进（AI、支付、TTS、文件、微信登录）。

### 3.4 基础设施层（Core）

- `core/config.py`：集中环境配置。
- `core/database.py`：异步数据库连接与会话管理。
- `core/security.py`：JWT 生成与校验。
- `main.py`：应用启动、CORS、静态文件挂载（`/uploads`）、路由注册与健康检查。

## 4. 移动端模块（`app/`）

### 4.1 页面层（Screens）

- `HomeScreen`：入口与概览。
- `QuestionnaireScreen`：问卷交互（当前以内置题目为主）。
- `ReportScreen`：报告主页面，支持：
  - 报告拉取、分析结果解析（JSON / 文本兜底）
  - 解锁状态分层展示（免费摘要 vs 完整内容）
  - 报告语音生成、播放/暂停、语音链接分享
  - 文字分享与微信分享预留入口
- `ChatScreen`：AI 聊天页面雏形（当前含模拟响应逻辑）。

### 4.2 API 封装层

- `src/api/client.ts`：统一请求封装，支持 Bearer Token。
- `src/api/reports.ts`：报告详情读取。
- `src/api/tts.ts`：报告语音生成。

### 4.3 客户端服务层

- `src/services/reportShare.ts`：文字报告生成、系统分享、语音播报辅助。
- `src/services/wechat.ts`：微信初始化、分享与支付调用封装（集成预留）。

### 4.4 导航壳层

- `App.tsx`：底部 Tab 导航（首页/问卷/报告/AI 助手）。

## 5. 小程序组件库模块（`components/`）

组件库提供适老化的报告展示部件：

- `health-summary`：健康总结卡片（含语音事件）。
- `abnormal-list`：异常指标列表（含点击事件）。
- `normal-list`：正常指标列表。
- `lifestyle-advice`：生活建议卡片（语音/分享/追问事件）。

并附 `pages/demo` 演示页用于本地联调与交互示例。

## 6. 目前功能完成度判断

从代码现状看，该项目已具备“可跑通的主链路骨架”：

- 已具备：登录、上传、问卷、分析、报告查看、支付解锁、追问上限控制、TTS 生成。
- 正在完善/待对接：
  - 部分移动端页面仍有 mock 或预留逻辑（如 Chat 模拟回复、微信支付/分享实连）。
  - 支付回调验签注释中明确提示为示例级实现，生产需升级为官方证书校验流程。
  - 报告解析与 UI 交互仍可继续细化（错误态、重试、轮询体验、更多指标可视化）。

整体上，项目已经形成“后端能力较完整 + 前端逐步对齐后端能力 + 小程序组件可复用”的三层结构。
